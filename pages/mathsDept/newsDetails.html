Here's an (intentionally brittle) algorithm for finding the postal code in HTML markup: If you find a paragraph with two tags, the postal code is the second word after the first comma in the second break tag. Although this algorithm works with this example, there are any number of perfectly valid addresses worldwide for which this simply wouldn't work. Even if you could write an algorithm that found the postal code for any address written in HTML, there are any number of paragraphs with two break tags that don't contain addresses at all. Writing an algorithm that looks at any HTML paragraph and finds any postal codes inside it would be extremely difficult, if not impossible.
